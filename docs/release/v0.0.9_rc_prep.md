## Asset Wave Summary

Comprehensive visual, audio, and automation layer for family-focused screen-time management.

### Mascot: Coachy (Tier-1 Parametric SVG)
- **20-state animation matrix**: Happy, neutral, sad, listening, thinking, celebrating, confused, approving, rejecting, warning, focused, proud, tired, curious, excited, sleepy, anxious, relieved, disappointed, triumphant
- **Parametric design**: Color swatch (light/dark/auto), size (24px–256px), emotion state + optional animation
- **Usage**: App icon, onboarding, reward celebrations, penalty warnings, feedback moments, marketing assets

### Icon System (63 Glyphs)

**Delivery formats:**
- Sprite sheet (single 2048×2048 PNG, optimized)
- Per-icon PNG (16×, 24×, 32×, 64×, 128×, 256×)
- SVG source (vector, scalable to any size)
- iOS asset catalog integration (AppIcon, UIImage sets)
- Android drawable resources (res/drawable, res/drawable-hdpi, etc.)

### Motion Pipeline

**Rive Sequences (12 looping):**
1. Breathing (idle loop) — 3s cycle, calm
2. Gesture-thinking — hand-to-chin, thoughtful
3. Gesture-listening — leaning forward, attentive
4. Success-celebration — jump + spin, jubilant
5. Error-shake — left/right motion, concerned
6. Loading-spinner — circular progress indicator
7. Transition-fade-in — entrance animation
8. Transition-fade-out — exit animation
9. Focus-entered — entrance flash + glow
10. Focus-exited — exit fade + dissolve
11. Reward-shower — confetti/particle effect
12. Penalty-warning — flash + rumble

**Lottie Fallbacks (12 rendered):**
- Same 12 sequences; JSON+PNG atlas for web/Android
- Size-gated at runtime: device RAM < 4GB → static PNG key frame; ≥4GB → full animation

**Haptic Mapping:**
- Each animation has paired haptic intensity curve
- Intensity levels: quiet (10%), medium (50%), intense (100%)
- Example: Success-celebration → intense haptic burst; breathing → subtle periodic taps

### Audio Cue Library

**Synthesized Cues (8, deterministic WaveTable):**
| Cue | Frequency (Hz) | Duration (ms) | Use Case |
|-----|----------------|---------------|----------|
| notification-ping | 800 + 1200 Hz | 200 | Incoming message, app opened |
| reward-success | 523 Hz sustained | 300 | Goal achieved, streak milestone |
| penalty-warning | 400 Hz falling | 250 | Time limit approaching, usage spike |
| timer-start | 523 Hz triplet | 150 | Timer began |
| timer-complete | 1047 Hz triplet | 400 | Timer ended, focus session complete |
| timer-tick | 300 Hz beep | 50 | Each second tick (opt-in, minimal) |
| focus-entered | 659 Hz rising | 200 | Focus session locked |
| focus-exited | 523 Hz falling | 200 | Focus session released |

**AHAP Patterns (8, Apple Haptic Audio Pattern):**
1. **success-tap** — sharp single tap (confidence feedback)
2. **warning-pulse** — 3× medium pulses (alert escalation)
3. **info-briefing** — 2× light taps (notification arrival)
4. **penalty-hit** — heavy impact + decay (consequence feedback)
5. **focus-lock** — firm sustained grip (immersion start)
6. **focus-unlock** — gentle release (immersion end)
7. **reward-shimmer** — 6× rapid light taps (celebration)
8. **error-buzz** — continuous buzz 100ms (error state)

**Implementation:**
- iOS: `CoreHaptics` (iOS 13+) + `AVAudioEngine` for cue playback
- Android: `VibrationEffect` for haptic patterns; `MediaPlayer` for audio cues
- Web: Web Audio API (synthesis) + Vibration API
- Rust source: `focus-audio` crate with deterministic WaveTable synthesis
