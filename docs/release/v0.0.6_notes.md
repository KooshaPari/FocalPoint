# FocalPoint v0.0.6 Release Notes

**Date:** 2026-04-23  
**Status:** Shipping — workspace clean, all 28 commits integrated

---

## Summary

FocalPoint v0.0.6 is a massive parallel-dispatch wave shipping 15+ features across Android, iOS, web, core, and infrastructure. All compilation failures from v0.0.5 have been resolved. The workspace compiles cleanly. Community-ready with full connector ecosystem (6 OAuth sources), governance framework in place, and comprehensive documentation.

---

## What's New

### Mobile Platforms

#### Android (NEW)
- **3 fully-functional Compose screens**: Today tab (daily view), Tasks (CRUD + swipe-to-delete), FocusTimer (countdown + haptics)
- Material Design 3 components with native navigation
- SQLite sync integration
- Ready for internal beta testing

#### iOS
- **iMessage/iCloud sync ready**: Apple Watch companion design + CRDT sync strategy published
- **Widget improvements**: Read-only SQLite bridge for Credits + TodayBrief in lock-screen widgets
- **In-process MCP**: iOS can now call focus-mcp-server over Unix-domain socket (opt-in)
- **StoreKit v2 server verification**: JWS verifier worker deployed; iOS client integrated

### Core Features

#### Connectors (6 Sources)
- **Apple Health**: HealthKit event stream ingestion (native iOS)
- **Fitbit**: OAuth2 + token persistence
- **Strava**: OAuth2 + activity import
- **Readwise**: Document highlights → focus-audit ingestion
- **Notion**: Database sync → task/event extraction
- **Linear**: Issue tracking → task inbox
- All connectors use standardized EventMapping trait; tokens persist via keychain

#### Graph Builder
- **Graph validator** with cycle detection and node-type cardinality rules
- **Save/load** with JSON round-trip
- **5 starter samples** (Morning Brief, Evening Shutdown, Gym Routine, Deep Focus, Bedtime Wind-Down)
- **Keyboard shortcuts** (Enter to create, Delete to remove, Tab to focus next)
- **IR export** for rule evaluation

#### UI Enhancements
- **Docs-site search**: Full-text search across all published docs
- **SEO + sitemap**: 35+ docs indexed; broken-link checker integrated
- **404 page** with smart suggestions
- **Sidebar reorganization**: Guides, References, Architecture clearly separated
- **Mascot preview**: CoachyDebugView shows pose + emotion + audio playback + haptics test button

### Developer Experience

#### Icon Generation
- **focus-icon-gen**: Procedural Coachy icon generator
- Automatic appiconset generation for iOS (@1x, @2x, @3x)
- Used by CI to keep visual assets in sync with codebase

#### Internationalization
- **Spanish translations**: 122 user-visible strings (es)
- **Japanese translations**: 122 user-visible strings (ja)
- Extraction from all Text, Label, Section, and accessibility components
- Ready for translator onboarding

#### Templates & Packaging
- **ed25519 manifest signing**: `manifest.toml` signed with private key
- **Pack format spec**: ZIP + metadata + signature schema documented
- **CLI install**: `focus template install <url>` downloads, verifies, unpacks

#### Observability
- **Sentry integration**: Error tracking + session replay with PII filtering
- **beforeSend filter**: Scrubs email, UUID, API keys before transmission
- **Dev test button**: Settings → Send test crash (validates integration)

#### Performance
- **Criterion benchmarks**: IR hash, eval tick, audit verify, starlark compile
- Baselines established for regression testing

### Infrastructure & Governance

#### Security
- **cargo-deny**: Dependency vulnerability scanning (advisories + supply-chain)
- **CycloneDX SBOM**: Software bill of materials for compliance audits
- **Dependency policy**: Pinned versions, audit cadence defined

#### Testing & Traceability
- **FR coverage matrix**: 5+ functional requirements backfilled with tests
- **Orphan trace cleanup**: Removed stale test references
- **Test discovery tool**: Scans codebase for `// Traces to: FR-*` markers

#### Documentation
- **Per-crate README**: All 20 crates document their purpose, public API, examples
- **Directory-level index**: `crates/README.md` with crate dependency graph
- **Threat modeling**: STRIDE analysis + security doc index
- **v2 Roadmap**: 18-month horizon (Phase 0–4, 26 FRs, capability-staged)

#### Governance
- **Issue templates**: Bug report, feature request, security disclosure

- **CODEOWNERS**: Ownership rules for `/crates/`, `/apps/ios/`, etc.

## Upgrade Notes

### For Testers
1. Clone `research-docs-push` or sync `main`
2. `cargo test --workspace` — all tests pass
3. iOS: `cd apps/ios && pod install && xcode apps/ios/FocalPoint.xcodeproj`
4. Android: `cd apps/android && ./gradlew assembleDebug`
5. Web builder: `cd apps/builder && bun install && bun run dev`

### For Contributors
- Check `docs/guides/quick-start/CONTRIBUTING.md` for setup
- Reference `docs/reference/honest_coverage.md` for feature status
- File bugs against GH Issues; feature requests reference corresponding FR (e.g., FR-FOCUS-005)
