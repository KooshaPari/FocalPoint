# FocalPoint v0.0.7 Release Notes

**Release Date:** 2026-04-24  
**Commits Since v0.0.6:** 141  
**Status:** Release Candidate

## Headline

v0.0.7 ships **org-audit integration**, **template collections**, and **phenotype-tooling lift**. The workspace consolidates cross-project infrastructure patterns and prepares for advanced authentication and team features in v0.0.8.

---

## Major Features

### Org-Audit Integration Framework
- **phenotype-bus (shared event bus)**: Unified event model across all services. Deduplication via canonical-hash + TTL + bloom filter. Webhook ingress for external event capture.
- **Admin sync orchestrator**: Coordinates multi-connector sync, rates, and conflict resolution. Per-connector state tracking and resumable batches.
- **Rule suggestion engine**: Heuristic pattern detection from audit chain history. Local inference, no external dependencies.

### Template Collections & Marketplace
- **Collections system**: 7 signed starter packs (Eidolon, Sidekick, Observably, Stashly, Paginary, + 2 community samples).
- **Trust graph**: ed25519 manifest signing, pack format spec, submission workflow.
- **Community submission portal**: Self-service template registration with moderation queue.

### Plugin-SDK Phase 2
- **Transport layer**: HTTP/SSE/WebSocket (in addition to STDIO). Bearer auth, rate-limit middleware, connection pooling.
- **Capability manifest**: Standardized schema for plugin capabilities (events, actions, queries, mutations).
- **hello-connector sample**: Fully functional connector archetype with auth flow, event mapping, and lifecycle hooks.

### GraphQL Gateway (Production-Ready)
- **Query surface**: Rules, tasks, audit history, connector status, wallet/penalty views.
- **Mutation surface**: Create/update/delete rules and tasks, trigger actions, manage connectors.
- **Subscription surface**: Real-time rule fires, task completions, balance changes.
- **Auth + rate-limiting**: JWT validation, per-client quotas, adaptive backoff.

### Language & DSL Enhancements
- **focus-lang**: 8 new high-level macros (reward, penalize, remind, celebrate, block, unlock_after, track_streak, if_pattern).
- **focus-replay**: Time-travel debugger for exploring alternate ruleset outcomes with diff reports.
- **VSCode extension**: FPL syntax highlighting, snippets, compile and preview commands.
- **Transpilers**: IR↔FPL, IR↔CLI reverse codegen. Starlark→IR compiler. 4 round-trip transpilers + property testing.

### Observability & Metrics
- **Tracing**: OpenTelemetry spans wired into sync, eval, audit, webhook, MCP, and CLI layers.
- **Metrics**: Prometheus counters/histograms for rule fires, action triggers, connector syncs, and storage operations.
- **Integration**: Sentry SDK + PII redaction, local opt-in telemetry + anonymous usage events.

### Testing & Verification
- **iOS E2E**: CoreHolder FFI harness with real SQLite + real FFI, 6 smoke scenarios.
- **Fuzz targets**: cargo-fuzz differential property tests with nightly CI.
- **Criterion baselines**: Established for IR hash, eval tick, audit verify, Starlark compile performance.

## Upgrade Path

1. **No migrations required.** SQLite schema stable since v0.0.4.
2. **Events dedup**: Existing audit records automatically deduplicated via canonical-hash + bloom filter (zero user action).
3. **Collections**: Marketplace URLs injected at runtime; no local user data changes.

## Performance Notes

- **Rule evaluation**: Criterion baseline shows ~500µs per eval tick (including audit write).
- **Event dedup**: Bloom filter + canonical-hash provides O(1) lookup with <1% false-positive rate.
- **CloudKit sync**: Incremental sync with server cursors; typical full sync <2s on LTE.
