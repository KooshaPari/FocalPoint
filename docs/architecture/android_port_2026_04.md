**Status:** Deferred Phase 2 scaffold (2026-04-23). iOS Phase 1 must ship first.  
**Target:** Android 14+ (API 34+); minimum API 29 for usability of accessibility + screen time APIs.

---

## 1. FFI Binding Strategy: UniFFI Kotlin

### Decision: UniFFI Kotlin (Rust → Kotlin via generate-bindgen)

We choose **UniFFI Kotlin** over alternatives:

| Option | Verdict | Why |
|--------|---------|-----|
| **UniFFI Kotlin** | ✅ CHOSEN | Same Mozilla-maintained toolchain as iOS (Swift FFI), async-aware coroutines, in Phenotype's supported matrix, proven on v0.28 |
| `uniffi-bindgen-kotlin-multiplatform` (fork) | ⚠ Secondary | Adds multiplatform support; useful for future Kotlin Multiplatform Mobile (KMM); not needed for v1 |
| `cxx-jni` (cxx crate bridge) | ❌ Rejected | Designed for C++, not Rust; requires C++ interop layer; adds complexity vs UniFFI |
| `gobley` / `babble` (JNI codegen) | ❌ Rejected | Unmaintained; no async support; brittle FFI contracts |

**Rationale:**
- UniFFI is Mozilla's official Rust ↔ {Swift, Kotlin, Python, Ruby} FFI generator — stable on v0.28.0.
- Already integrated in `crates/focus-ffi` (see `Cargo.toml` `uniffi = "0.28"`).
- Generates **coroutine-aware Kotlin interfaces** (async functions → suspend funs).
- Build script can extend iOS strategy: same `uniffi-bindgen` CLI, different output language.
- Bindings emit as `*.kt` source files → packaged into `.aar` (Android Archive) → consumed by app's `build.gradle.kts`.

## 2. Android Architecture

### App Structure

Mirror iOS tabs in Jetpack Compose + Material 3:

| Screen | Purpose | Platform-Specific Notes |
|--------|---------|---|
| **Tasks** | Upcoming assignments / to-dos (from connectors) | Canvas OAuth via Custom Tabs; same connector intake as iOS |
| **Focus** | Active session timer + real-time rule evaluation | Uses Android `Handler.postDelayed` + local broadcast; same `FocalPointCore` evaluator |
| **Today / Rituals** | Daily dashboard + habit tracking | Placeholder for future ritual templates |
| **Rules** | Rule builder + visual editing (read-only v1) | Read-only rule browser; editing deferred to Phase 3+ |
| **Wallet** | Credits balance + transaction log | Same `focus-rewards::WalletStore` via FFI |
| **Activity** | Audit log viewer (read-only) | Readonly audit chain display; write via Rust layer |
| **Settings** | Permissions, accounts, diagnostics | Android-specific: toggle AccessibilityService, revoke UsageStats, sign out |

**Mascot:** Coachy (Lottie animations v1; Rive deferred to Phase 2+).

### Core FFI Layer

**File:** `app/src/main/kotlin/com/focalpoint/ffi/FocalPointCore.kt`

Generated by `uniffi-bindgen`:
- `class FocalPointCore(storagePath: String, userId: String)`
- Public interface mirrors Rust `FocalPointCore` struct:
  - `suspend fun evaluateRules(appId: String): RuleFireResult?`
  - `suspend fun logEventBatch(events: List<EventRecord>): Unit`
  - `suspend fun getWalletBalance(): WalletState`
  - `suspend fun getAuditRecords(limit: Int): List<AuditRecord>`

No Kotlin-side domain logic. All state is in Rust. Kotlin is a thin adapter.

### Enforcement: Android Equivalents

| iOS Component | Android Equivalent | File |
|---|---|---|
| **ManagedSettings framework** | AccessibilityService + UsageStatsManager | `enforcement/AndroidEnforcementDriver.kt` |
| **Application blocking** | (AccessibilityService can't block; workaround via DevicePolicyManager or overlay) | See "Technical Gaps" below |
| **Screen Time query** | `UsageStatsManager.queryAndAggregateUsageStats()` | `analytics/UsageStatsAdapter.kt` |
| **Notification of enforcement** | Toast + Foreground Service notification | `enforcement/EnforcementNotifier.kt` |

**EnforcementDriver trait (Rust):**
```rust
pub trait EnforcementDriver {
    async fn enforce(&self, rule_id: RuleId) -> Result<()>;
    async fn revoke(&self, rule_id: RuleId) -> Result<()>;
}
```

**Android impl (Kotlin):**
- Receives rule from Rust evaluator.
- Queries AccessibilityService to detect if blocked app is in foreground.
- If yes: show full-screen overlay + haptic feedback + play explanation audio (if configured).
- Log enforcement event back to Rust audit chain.

### Storage

**Decision: Reuse Rust SQLite path.**

- `FocalPointCore` uses same `focus-storage::sqlite::SqliteAdapter`.
- App provides path: `getFilesDir().absolutePath + "/core.db"`.
- No Room (Android Jetpack persistence library). Rust is the source of truth; Kotlin reads via FFI.
- Benefits: single schema, single audit trail, single event log.

### Push Notifications & Sync

**v1 (MVP): Local-first, no FCM.**
- WorkManager periodic job (equivalent to iOS BGTaskScheduler) fires every 15 min.
- Job calls `evaluateRules()` on cached app list; no network until connector sync needed.
- Scheduler: see `scheduler/WorkManagerSyncScheduler.kt` (deferred to Phase 2).

**v1.5+ (optional Phase 5):** FCM for instant notifications (out of scope).

### Background Service Architecture

**Android constraint:** Foreground services require persistent notification to keep process alive.

- `FocalPointEnforcementService` (extends `Service`) stays active when rules are evaluated.
- Notification: "FocalPoint is monitoring" (Material 3 neutral color).
- WorkManager schedules periodic wake-ups if the service is killed (rare but handles app kills).

## 3. Android-Specific Gaps & Technical Decisions

### Gap 1: App Blocking (No ManagedSettings Equivalent)

**Problem:** Android 14+ removed the ability for third-party apps to directly block other apps' execution. `ManagedSettings` (Apple) → no Android equivalent.

**Solution for v1:**
- **Overlay approach:** Display full-screen "Go focus!" overlay that consumes touch input while blocked app is in foreground.
- AccessibilityService logs app lifecycle (detected via `AccessibilityEvent.TYPE_WINDOW_STATE_CHANGED`).
- On detection, launch overlay Activity (non-finish-able until time expired or user completes a challenge).

**Long-term (v2+):**
- Require device admin status → use `DevicePolicyManager.lockNow()` (nuclear option, rarely used).
- Or partner with Android's "Digital Wellbeing" integration (requires OEM partnership).

**Tradeoff:** v1 enforcement is "soft" (overlay + notification) not "hard" (locked). User can force-quit the overlay. Trade documented; acceptable for Phase 1 as long as it's transparent.

### Gap 2: Screen Time / Usage Stats

**Solution:** `UsageStatsManager.queryAndAggregateUsageStats(startTime, endTime)` returns app foreground duration.

- Kotlin adapter queries this every 15 min.
- Passes aggregate data to Rust evaluator.
- Maps to our `ActivityRecord` domain type.

**Caveat:** Requires `PACKAGE_USAGE_STATS` permission (user grant, not install-time; deeper in Settings > Apps > Special Permissions).

### Gap 3: Notification Variants (Android 14+)

**iOS equivalent:** Live Activities on lock screen → "focus timer is running".

**Android equivalent:** Notification foreground types + Wear OS Ongoing Activity (Phase 2+).

For v1: Standard persistent notification in notification drawer (sufficient).

## 5. Bindings Generator: Rust Binary

**File:** `crates/focus-ffi/src/bin/android_bindings.rs`

### Strategy

Instead of a shell script (which violates Phenotype scripting policy), use a small Rust binary that:
1. Validates Android NDK is installed (`ANDROID_NDK_HOME` env var).
2. Runs `cargo build --release -p focus-ffi --target <ABI>` for all 4 Android ABIs.
3. Runs `uniffi-bindgen generate` with `--language kotlin` output.
4. Packs `.so` libraries into `apps/android/app/src/main/jniLibs/<abi>/`.

### Implementation

```rust
// crates/focus-ffi/src/bin/android_bindings.rs
use anyhow::{anyhow, Result};
use std::env;
use std::path::{Path, PathBuf};
use std::process::Command;

fn main() -> Result<()> {
    let ndk_home = env::var("ANDROID_NDK_HOME")
        .map_err(|_| anyhow!("ANDROID_NDK_HOME not set. Install Android NDK and export the path."))?;
    
    let workspace_root = find_cargo_root()?;
    let ffi_crate = workspace_root.join("crates/focus-ffi");
    let android_app = workspace_root.join("apps/android/app");

    // Target triples for all Android ABIs
    let targets = vec![
        "aarch64-linux-android",      // ARM64 (primary)
        "armv7-linux-androideabi",    // ARM32 (some devices)
        "x86_64-linux-android",       // Intel64 (emulator)
        "i686-linux-android",         // Intel32 (legacy)
    ];

    eprintln!("Building focus-ffi for Android targets...");

    for target in &targets {
        eprintln!("  → {}", target);
        let status = Command::new("cargo")
            .arg("build")
            .arg("--release")
            .arg("-p")
            .arg("focus-ffi")
            .arg("--target")
            .arg(target)
            .current_dir(&workspace_root)
            .status()?;

        if !status.success() {
            return Err(anyhow!("cargo build failed for {}", target));
        }
    }

    eprintln!("Generating Kotlin bindings...");
    let status = Command::new("cargo")
        .arg("run")
        .arg("--release")
        .arg("-p")
        .arg("focus-ffi")
        .arg("--bin")
        .arg("uniffi-bindgen")
        .arg("--")
        .arg("generate")
        .arg(ffi_crate.join("src/focus_ffi.udl"))
        .arg("--language")
        .arg("kotlin")
        .arg("--out-dir")
        .arg(android_app.join("src/main/kotlin/com/focalpoint/ffi"))
        .current_dir(&workspace_root)
        .status()?;

    if !status.success() {
        return Err(anyhow!("uniffi-bindgen failed"));
    }

    eprintln!("Packing .so libraries into jniLibs...");
    let abi_map = vec![
        ("aarch64-linux-android", "arm64-v8a"),
        ("armv7-linux-androideabi", "armeabi-v7a"),
        ("x86_64-linux-android", "x86_64"),
        ("i686-linux-android", "x86"),
    ];

    for (target, abi) in abi_map {
        let so_src = workspace_root
            .join("target/release")
            .join(target)
            .join("libfocus_ffi.so");
        let so_dst = android_app
            .join("src/main/jniLibs")
            .join(abi)
            .join("libfocus_ffi.so");

        std::fs::create_dir_all(so_dst.parent().unwrap())?;
        std::fs::copy(&so_src, &so_dst)
            .map_err(|e| anyhow!("Failed to copy {} to {}: {}", so_src.display(), so_dst.display(), e))?;
    }

    eprintln!("✅ Android bindings generated successfully.");
    Ok(())
}

fn find_cargo_root() -> Result<PathBuf> {
    let mut current = std::env::current_dir()?;
    loop {
        if current.join("Cargo.toml").exists() {
            return Ok(current);
        }
        if !current.pop() {
            return Err(anyhow!("Could not find Cargo workspace root"));
        }
    }
}
```

### Invocation

```bash
# From workspace root:
cargo run --release -p focus-ffi --bin android_bindings
```

**Error handling:**
- Fails loudly if `ANDROID_NDK_HOME` unset (no silent skip).
- Fails if any cargo build step fails.
- Fails if bindgen step fails.
- Fails if .so copy fails.

#### `CoreHolder.kt`

```kotlin
package com.focalpoint.core

import android.content.Context
import androidx.lifecycle.ViewModel
import com.focalpoint.ffi.FocalPointCore
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.launch

class CoreHolder(val context: Context) : ViewModel() {
    private lateinit var core: FocalPointCore

    init {
        val storagePath = context.filesDir.absolutePath
        CoroutineScope(Dispatchers.IO).launch {
            core = FocalPointCore(storagePath, "default-user")
        }
    }

    suspend fun getWalletBalance() = core.getWalletBalance()
    suspend fun evaluateRules(appId: String) = core.evaluateRules(appId)
    suspend fun getAuditRecords(limit: Int) = core.getAuditRecords(limit)

    companion object {
        private var instance: CoreHolder? = null

        fun getInstance(context: Context): CoreHolder {
            return instance ?: synchronized(this) {
                CoreHolder(context).also { instance = it }
            }
        }
    }
}
```

## 7. Enablement & First-Run

**Prerequisites:**
1. **Android NDK**: Download via Android Studio → SDK Manager → SDK Tools → "NDK (Side by side)".
   - Path: `/Users/you/Library/Android/sdk/ndk/<version>` (macOS).
   - Set: `export ANDROID_NDK_HOME=/Users/you/Library/Android/sdk/ndk/<version>` in `.bashrc` / `.zshrc`.

3. **Rust targets**: Install via `rustup`:
   ```bash
   rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android
   ```

### First Run

1. **Generate bindings:**
   ```bash
   cd /Users/you/CodeProjects/Phenotype/repos/FocalPoint
   cargo run --release -p focus-ffi --bin android_bindings
   ```
   - Creates `apps/android/app/src/main/kotlin/com/focalpoint/ffi/FocalPointCore.kt`.
   - Copies `.so` libs to `apps/android/app/src/main/jniLibs/`.

2. **Open in Android Studio:**
   ```bash
   open -a "Android Studio" apps/android
   ```
   - Gradle sync auto-detects Kotlin DSL, downloads dependencies.
   - First build takes ~3 min (download Gradle + SDK dependencies).

3. **Run on device/emulator:**
   ```bash
   cd apps/android && ./gradlew installDebug
   ```
   - If no device is connected: open Android Studio → Device Manager → Launch emulator first.

4. **Expected behavior (v0.0.1):**
   - App launches.
   - Placeholder banner visible on all screens.
   - Tapping "Wallet" tab shows credits balance (read from Rust via FFI).
   - No enforcement; no connectors; no rules configured.

## 8. Major Unknowns & Deferred Decisions

### Unknown 1: App Blocking UX (Soft vs Hard)

**Issue:** Android forbids third-party app-blocking as of Android 13+.

**Decision for v1:** Use overlay + notification (soft blocking).

**Deferred:** DevicePolicyManager integration (v2+) requires device admin, which is a friction point for onboarding. Acceptable for beta; revisit after user feedback.

### Unknown 2: Accessibility Service Lifetime

**Issue:** AccessibilityService can be killed by the system if memory is tight; no guarantee it persists.

**Decision for v1:** WorkManager job re-enables it if killed.

**Deferred:** Foreground service pinning (requires `android.permission.FOREGROUND_SERVICE`) is available in Android 12+ but complicates manifest.

### Unknown 3: Wear OS Ongoing Activity (Live Activities equiv)

**Issue:** iOS Live Activities (lock screen timer) → Android Wear OS Ongoing Activity (smartwatch persistent notification).

**Decision for v1:** Skip (not all users have Wear OS).

**Deferred to Phase 2+:** Wear app + Ongoing Activity integration.

### Unknown 4: Material You Dynamic Color

**Issue:** Android 12+ supports Material You theming (device color palette extraction).

**Decision for v1:** Hardcoded Material 3 palette (Indigo primary, Purple secondary, Amber tertiary).

**Deferred:** Dynamic color support (phase 2+).
