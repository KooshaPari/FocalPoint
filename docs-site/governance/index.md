# Governance

FocalPoint is open-source software governed by transparent policies for contributions, security, and community participation.

## Core Documents

- **[Contribution Guide](./contributing)** — How to submit PRs, build locally, run tests
- **[Code of Conduct](./coc)** — Community standards and expected behavior
- **[Verification Criteria](./verification)** — Security and functional requirements for connectors & rule packs

## Contributing

FocalPoint welcomes contributions in many forms:

- **Code**: Bug fixes, new features, refactoring
- **Documentation**: Guides, API docs, examples
- **Connectors**: New integrations with external platforms
- **Rule packs**: Curated rule templates for common use cases
- **Translation**: Localize FocalPoint for new languages
- **Testing**: Report bugs, write test cases, conduct security reviews

See [Contribution Guide](./contributing) for the full process.

## Security Policy

All security issues should be reported **privately** to security@focalpoint.local (or via [GitHub Security Advisory](https://github.com/KooshaPari/FocalPoint/security/advisories)).

See [Verification Criteria](./verification) for security requirements for published connectors and rule packs.

## Decision-Making

Major design decisions are recorded as **Architectural Decision Records (ADRs)** in [Architecture](../architecture/adrs).

For proposed changes:

1. Open an issue or discussion
2. Post a draft ADR to [docs/adr/](../architecture/adrs)
3. Gather feedback from maintainers and community
4. Merge once consensus is reached

## Community Standards

FocalPoint follows a **Contributor Covenant Code of Conduct**. See [Code of Conduct](./coc) for details.

We are committed to:

- **Inclusivity**: Welcome contributors of all backgrounds and experience levels
- **Respect**: Treat each other with kindness and professionalism
- **Transparency**: Communicate openly about decisions, blockers, and trade-offs
- **Accountability**: Hold ourselves to high standards of behavior and quality
